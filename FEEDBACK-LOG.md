## 2026-06-14 ② 結晶記事の冒頭が、自分の本文と矛盾していた

- **対象**: `2026-06-04-crystallographic-restriction.md`（結晶に五角形がない理由）
- **指摘**: 「7回対称や8回対称も無いのだから、『ひとつだけ"絶対に現れない対称"がある』という冒頭の記述はおかしい。5回だけが特別というのは変では」
- **正しい事実**: 周期的な（並進対称を持つ）結晶で許される回転対称は **1・2・3・4・6 回だけ**。5回も、7・8・9…回も禁制。しかも 5回対称は「絶対に」現れないわけでもない ── **準結晶**は周期性を捨てることで 5回対称を持つ。
- **原因**: 本文はもともと正しかった（7・8回も禁制、準結晶の話まで書いてある）。誤っていたのは**冒頭の掴みの一文だけ**。「ひとつだけ絶対に現れない」と言い切ったせいで、自分の本文と矛盾していた。
- **対応**: 冒頭を「現れない回転対称もある。なかでも有名なのが5回」に修正。5回を「唯一・絶対の禁制」ではなく「いちばん小さく、いちばん有名な禁制（しかも準結晶で唯一復活した）」として立て直した。
- **教訓**: **冒頭の掴みで言い切りすぎて、本文と矛盾させない。** 多くの読者は冒頭に反応する。物理の禁制は前提つき（「周期的な結晶では」）で述べる。本文が正しくても、掴みが過剰だと指摘は来る。

## 2026-06-14 ① 数式（コードブロック）が暗背景に暗文字で読めなかった

- **対象**: `2026-06-14-lock-in-amplifier.md`（をはじめ、コードブロックを使う全記事）
- **指摘**: 「数式が非常に見づらい。文字色を白にするなど、背景色に対しコントラストをつけるべき」
- **原因**: ```で囲った数式に、Astro 既定の構文ハイライト（Shiki, テーマ `github-dark`）が**暗い背景**を当てる。一方サイトの CSS は `pre > code { color: var(--text) }` で文字を**暗色 (#1d1b17)** に固定していた。結果「暗背景に暗文字」でほぼ不可視。サイト本体は warm light（`--card: #fff`）なのに、コードブロックだけ暗かった。
- **対応**: `astro.config.mjs` の `markdown` に **`syntaxHighlight: false`** を追加。Shiki の暗い inline スタイルが消え、サイト本来の白背景＋黒文字に戻る。1記事ではなく**全記事**にまとめて効く。
- **検証**: ビルドして dist をスクリーンショットで目視。コードブロック内の数式が黒文字で読めることを確認。
- **教訓**: コードブロックを数式に使う構成では、**レンダリング後のコントラストを実際に目で見る**。テーマの既定値（暗テーマ）とサイトの配色がぶつかっていないか。
